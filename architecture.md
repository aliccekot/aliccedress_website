## Список разрабатываемых компонентов
### Библиотека общих компонентов (ui-library):
- **Button** - Универсальная кнопка с вариантами размеров, цветов и состояний
- **Input** - Поле ввода с поддержкой валидации, ошибок и различных типов
- **Modal** - Модальное окно для отображения форм, уведомлений и дополнительного контента
- **Card** - Карточка для структурированного отображения контента
- **Loader** - Индикатор загрузки для асинхронных операций
- **Navbar** - Навигационная панель с меню и ссылками
- **Breadcrumbs** - Навигационная цепочка для отображения пути страницы
- **Container** - Ограничивающий контейнер для выравнивания контента по ширине
- **Grid** - Сетка для адаптивного расположения элементов


### Основное приложение (main-app):
- **Header** - Шапка сайта с логотипом, навигацией и корзиной
- **Footer** - Подвал сайта с контактной информацией и ссылками
- **ProductCard** - Карточка товара для отображения в каталоге с изображением, названием и ценой
- **ProductList** - Список товаров с поддержкой сеточного отображения
- **ProductDetails** - Детальная информация о товаре с галереей изображений и описанием
- **CartItem** - Элемент товара в корзине с возможностью изменения количества
- **CartSummary** - Итоговая информация корзины с подсчетом стоимости
- **LoginForm** - Форма входа в аккаунт с валидацией данных
- **RegisterForm** - Форма регистрации нового пользователя


## Структура роутинга
### Публичные маршруты (доступны без авторизации):
- `/` - Главная страница (компонент Home)
- `/catalog` - Каталог товаров (компонент Catalog)
- `/product/:id` - Страница товара (компонент Product)
- `/auth/login` - Страница входа (компонент Auth/Login)
- `/auth/register` - Страница регистрации (компонент Auth/Register)


### Защищенные маршруты (требуют авторизации):
- `/profile` - Личный кабинет (компонент Profile)
- `/cart` - Корзина товаров (компонент Cart)
- `/checkout` - Оформление заказа (компонент Checkout)


### Логика защиты маршрутов:
- При попытке доступа к защищенным маршрутам выполняется проверка авторизации
- Неавторизованные пользователи перенаправляются на страницу `/auth/login`
- После успешной авторизации пользователь возвращается на запрашиваемую страницу
- Контекст авторизации (AuthContext) управляет состоянием аутентификации
